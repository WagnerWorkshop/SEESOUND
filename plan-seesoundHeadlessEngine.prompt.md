## Plan: SEESOUND v2.0 — Headless Web Engine with Quad-Brain Pipeline **TL;
DR** Refactor SEESOUND into a headless,
library-first web engine with a built-in default UI. Core is a dynamic dependency graph that lazily computes only what active rules consume. A Quad-Brain AudioWorklet (Pitch 16384, Texture 4096, Rhythm 1024) with sub-bin parabolic interpolation feeds Harmonic Objects in Entity Mode or raw magnitudes in Particle Mode. UI gains progressive disclosure,
tabs,
descriptions,
and a new-project-autosave flow. Long-term: migrate hot paths to Rust/WASM+WebGPU. --- ### Phase 1: Headless Core & Dependency Graph (Foundation) **Goal**: Separate engine from UI;
make all computation lazy and dependency-driven. 1. **Extract `SeesoundEngine` class** — Create `src/engine/SeesoundEngine.js` as the headless API surface. It owns the AudioEngine,

The graph emits a flat `AudioUsage` object consumed by AudioEngine (already has `setRuleInputUsage`). *Extends existing `_collectRuleInputs` logic in RuleCompiler.* 3. **Axiomatic-only computation** — In `DependencyGraph`,
mark variables as either `axiomatic` (always computed: `time`, `deltaTime`, `canvasWidthPx`, etc.) or `derived` (computed only if referenced). The engine skips all non-referenced derived computations at the frame level. 4. **Refactor `main.js`** — Strip to~50 lines: instantiate `SeesoundEngine`,
mount the default UI shell,

harmonic association,
and entity/particle mode switching. Remove user-facing FFT resolution controls. **Steps** (*depends on Phase 1 step 2*): 1. **Remove FFT resolution from UI** — Delete the `fftSize`/`audioResolution` slider from `SETTINGS_SLIDERS` in `ControlPanel.js`. Remove `ALLOWED_FFT_SIZES` from `AudioEngine.js`. Hardcode: Pitch=16384,
Texture=4096,
Rhythm=1024. 2. **Sub-Bin Parabolic Interpolation** — In `BinAnalysisWorklet._findFundamental()`,
after locating the peak bin index,
fit a parabola through the peak and its two neighbors: `truePeak=peakIndex+(mag[peak-1] - mag[peak+1]) / (2*(2*mag[peak] - mag[peak-1] - mag[peak+1]))`. Compute sub-bin frequency with `truePeak * sampleRate / n`. Return precise `fundamentalHz`. 3. **Harmonic Association (Comb Filter)** — In `BinAnalysisWorklet`,
after _findFundamental,
for each fundamental $f_0$,
iterate $k=2,
3,
4...$ up to Nyquist,
sample magnitude at $k \cdot f_0$,
accumulate harmonic energy into the fundamental's object. Emit `harmonicObjects[]` array with `{ fundamentalHz, pitchClass, volume, harmonicEnergy, centroid, flatness, inharmonicity }`.


4. **Entity Mode vs Particle Mode** — Add `objectMode` parameter (`'particle'` | `'entity'`). In particle mode: worklet emits `magnitude` Float32Array only (existing behavior). In entity mode: worklet runs full pitch+texture+rhythm pipeline,
emits `entityMetrics` with `harmonicObjects[]` array. *Already partially wired via `needPitchBrain`/`needTextureBrain`/`needRhythmBrain` flags.* 5. **Texture Brain** — For each harmonic object,
compute localized spectral flatness,
centroid,
and inharmonicity over its harmonic bins only (not globally). Emit `entityCentroid`,
`entityFlatness`,
`entityInharmonicity` per object. 6. **Rhythm Brain & Temporal Grouping** — In the 1024-window FFT,
detect `globalTransient` when broadband energy spikes above threshold. Group distant harmonic objects that share the same onset frame into temporal clusters. 7. **Tracker Brain** — Match harmonic objects frame-to-frame using nearest-fundamental with pitch-continuity. Assign persistent `streamId` and increment `entityAge`. **Relevant files**: - MODIFY: `frontend/src/engine/audio/BinAnalysisWorklet.js` — Parabolic interpolation,
comb filter,
harmonic objects - MODIFY: `frontend/src/engine/audio/AudioEngine.js` — Handle `entityMetrics` with object arrays,
remove fftSize - MODIFY: `frontend/src/engine/ControlPanel.js` — Remove FFT slider,
add objectMode toggle - MODIFY: `frontend/src/engine/ui/MenuSettings.js` — Remove fftSize slider,
add foresight/delay sliders **Verification**: 1. Play a pure sine tone,

3f_0,
...$. 3. Switch objectMode to entity,
verify `entityMetrics` messages arrive with `harmonicObjects`. 4. Switch to particle mode,
verify only `magnitude` array is sent. --- ### Phase 3: Rule Engine & UI Overhaul **Goal**: Remove luma,
